索引篇

索引是数据的目录。

索引常见面试题

  1. 索引的分类：

    • 按「数据结构」分类：B+ 数索引、Hash 索引、Full-text 索引。
    • 按「物理存储」分类：聚簇索引（主键索引）、二级索引（辅助索引）。
    • 按「字段特性」分类：主键索引、唯一索引、普通索引、前缀索引。
    • 按「字段个数」分类：单列索引、联合索引。

  2. 什么时候需要/不需要索引
    索引最大的好处是提高查询速度，但其缺点也很明显：

    • 需要占用物理空间，数量越多，占空间越大；
    • 创建和维护索引都需要耗费时间，这种时间随着数据量的增加而增大；
    • 会降低表的增删改效率，因为 B+ 树为了维护索引有序性，都需要进行动态维护。

    什么时候需要索引？

    • 字段有唯一性限制；
    • 经常用于 where 查询条件的字段；
    • 经常用于 group by 和 order by 的字段。

    什么时候不需要索引？

    • where、group by 和 order by 中不会用到的字段；
    • 字段存在大量重复值；
    • 表数据少。
    • 经常更新。

  3. 有什么优化索引的方法？

    • 前缀索引优化
    • 覆盖索引优化
    • 主键自增
    • 防止索引失效

从数据页的角度看 B+ 树

  • InnoDB 是如何存储数据的？
  • B+ 树是如何进行查询的？
  • 聚簇索引和二级索引

InnoDB 的数据是按‘数据页’为单位来读写的，默认数据页大小为 16KB。数据页之间通过双向链表的形式组织起来，物理上不连续，但是逻辑上连续。

数据页内包含用户记录，每个记录之间用单向链表的方式组织起来，为了加快在数据页内的查询效率，设计了一个页目录。页目录内存储各个槽（分组），且主键值是有序的，于是可以通过二分查找法的方式进行检索从而提高效率。

为了高效查询记录所在的数据页，InnoDB 采用 B+ 树作为索引，每个节点都是一个数据页。

如果叶子节点存储的是实际数据，则称之为聚簇索引，一个表只能有一个聚簇索引；如果叶子节点存储的不是实际数据，而是主键值，则称之为二级索引，一个表中可以由多个二级索引。

高性能服务器程序框架

发表于 2023-03-22 分类于 网络编程

服务器可以解构为以下三个主要模块：
IO处理单元：五种IO模型和两种高效事件处理模式。
逻辑单元：两种高效的并发模式，以及高效的逻辑处理方式——有限状态机
存储单元：作为服务器的可选模块，我们并不讨论。

服务器模型

C/S 模型

C/S 模型的逻辑很简单：

  • 服务器启动后，首先创建一个（或多个）监听 socket，并 bind 一个端口号，然后就调用 listen 函数坐等客户端的连接。
  • 服务器稳定运行后，客户端就通过调用 connect 函数向服务器发起连接。

由于客户端的连接请求是随机到达的异步事件，服务器需要使用某种 I/O 模型来监听这一事件。上图中，服务器使用的是 I/O 复用技术之一的 select 系统调用。

当监听到连接请求后，服务器就调用 accept 函数接受它，并分配一个逻辑单元为之服务。

逻辑单元可以是一个子进程、子线程或其它。上图中，服务器给客户端分配的逻辑单元是由 fork 系统调用创建的子进程。

逻辑单元读取客户请求，处理该请求，然后将处理结果返回给客户端。

socket 函数 —— 创建

函数原型

函数作用

socket函数用于创建套接字，也就是向系统申请一个socket资源，确定套接字的各种属性。函数调用成功使返回一个 socket 文件描述符，失败则返回 -1 并设置 errno。

参数说明

  1. domain
    该参数告知系统使用哪个底层协议族。就 TCP/IP 协议族而言，该参数可设置为 PF_INET（Protocol Family of Internet，用于 IPv4）或 PF_INET6（用于IPv6）。更多协议族可自行查阅源码。

  2. type
    该参数指定服务类型。服务类型主要有 SOCK_STREAM（流）和 SOCK_UGRAM（数据报）服务。对于 TCP/IP协议族而言，取值 SOCK_STREAM 表示传输层使用 TCP 协议，取 SOCK_UGRAM 表示传输层用 UDP 协议。
    在 Linux 内核版本 2.6.17 起，type 参数可以接受上述服务类型与下面两个重要标志相与的值：SOCK_NONBLOCK 和 SOCK_CLOEXEC。它们分别表示将新创建的 socket 设为非阻塞的，以及用 fork 调用创建子进程时在子进程中关闭该 socket。
    在此之前，文件描述符的这两个属性都需要使用额外的系统调用（如 fcntl）来设置。

  3. protocol
    该参数是在前两个参数构成的协议集合下，再选择一个具体的协议。不过这个值通常都是唯一的（前两个参数已经足够决定其值了）。几乎所有的情况，我们都把它设为 0，表示默认协议。

字节序

字节序分为大端字节序（big-endian）和小端字节序（little-endian）。大端字节序指的是一个整数的高位字节存储在内存的低地址处，低位字节存储在内存的高地址处；小端字节序恰好相反。

当格式化的数据在两台使用不同字节序的主机之间直接传递时，接收端必然会错误解释数据。解决的办法是：发送端总是把要发送的数据转化为大端字节序数据再发送，接收端知道来的数据是大端字节序，再根据自身情况转换数据。
因此大端字节序也被称为网络字节序，它给所有接收数据的主机提供了一个正确解释收到格式化数据的保证。

二叉搜索树（BST）

性质

一棵 BST 中的任意节点的值需要满足：

  1. 该节点的值不小于它左子树中任意节点的值。
  2. 该节点的值不大于它右子树中任意节点的值。
    显然，BST 的中序遍历是一个单调递增的序列。

BST 的检索

在 BST 中查找是否存在值为 x 的节点，执行以下过程：

  1. 如果节点 p 的值为 x，则找到。
  2. 如果节点 p 的值 >x：
    1. 如果节点 p 左孩子为空，则不存在 x；
    2. 否则在节点 p 的左子树中查找。
  3. 如果节点 p 的值 <x：
    1. 如果节点 p 右孩子为空，则不存在 x；
    2. 否则在节点 p 的右子树中查找。

BST 的插入

在 BST 中插入值为 x 的节点。与检索过程类似，如果发现要走向的子节点为空时，则新建值为 x 的节点作为子节点。
