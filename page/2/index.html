建议搭配视频一起食用：C语言线程池、C改C++版。

线程池的原理

我们使用线程时就去创建一个线程，这样实现起来确实简便，但实际工程中会有一个问题：如果并发的线程数量很多，并且每个线程都是执行了一个时间很短的任务就结束了，那么频繁创建线程就会大大降低系统的效率，因为线程的销毁和创建都需要时间。

那么有没有一种办法可以使得线程能够复用，即执行一个任务后不被销毁，而是可以继续执行其它任务呢？

线程池是一种多线程处理形式，处理过程中将任务添加到队列，在创建线程后自动启动这些任务。线程池线程都是后台线程，每个线程都使用默认的堆栈大小，以默认的优先级运行，并处于多线程单元中。如果某个线程在托管代码中空闲（如正在等待某个事件），则线程池将插入另一个辅助线程来帮助所有处理器保持繁忙。如果所有线程池的线程都始终保持繁忙，但队列中包含挂起的工作，则线程池将在一段时间后创建另一个辅助线程，但线程的数量永远不会超过最大值。超过最大值的线程可以排队，当它们要等到其它线程完成后才能启动。

在各个编程语言中都有线程池的概念，并且很多语言中直接提供了线程池，下面给大家介绍一下线程池的实现原理：

  • 线程池的组成主要分为 3 个部分：
    1. 任务队列，存储需要处理的任务，由工作的线程来处理这些任务
      • 通过线程池提供的API函数，将一个待处理的任务添加到任务队列，或者从任务队列中删除
      • 已处理的任务会从任务队列中删除
      • 线程池的使用者，也就是调用线程池函数往任务队列里添加任务的线程就是生产者线程
    2. 工作的线程（任务队列任务的消费者），N个
      • 线程池中维护了一定数量的工作线程，它们的作用是不停地读任务队列，从中取出任务并处理
      • 工作的线程相当于是任务队列的消费者角色
      • 如果任务队列为空，工作的线程将会被阻塞（使用条件变量\信号量阻塞）
      • 如果阻塞之后有了新任务，生产者将解除阻塞，工作线程开始工作
    3. 管理者线程（不处理任务队列中的任务），1个
      • 它的任务是周期地对任务队列中的任务数量及处于忙状态的工作线程个数进行检测
      • 当任务过多时，可以适当地创建一些新的工作线程
      • 当任务过少时，可以适当地销毁一些工作线程

线程同步概念

假设有4个线程 ABCD，当前一个线程 A 对内存中的共享资源进行访问时，其它线程 BCD 都不可以对这块内存进行操作，直到线程 A 对这块内存访问完毕为止，BCD 其中之一才能访问这块内存，剩余的两个需要继续阻塞等待，一次类推，直至所有的线程都对这块内存操作完毕。线程对内存的这种访问方式就称之为线程同步。
通过概念可知：所谓的同步并不是多个线程同时对内存进行访问，而是按照先后顺序依次进行的。

线程概述

线程是轻量级的进程（LWP：light weight process），在 Linux 环境下线程的本质仍是进程。
在计算机上运行的程序是一组指令及指令参数的组合，指令按照既定的逻辑控制计算机运行。操作系统会以进程为单位，分配系统资源，可以这样理解：进程是资源分配的最小单位，线程是操作系统调度执行的最小单位。

先从概念上了解一下线程和进程之间的区别：

  • 进程有自己独立地址空间，多个线程共用一个地址空间
    • 线程更加节省系统资源，效率可以持平甚至更高；
    • 在一个地址空间中多个线程独享栈区、寄存器（内核管理中的）；
    • 在一个地址空间中多个线程共享代码段、堆区、全局数据区、打开的文件（文件描述符表）。
  • 线程是程序的最小执行单位，进程是操作系统最小的资源分配单位
    • 每个进程对应一个虚拟地址空间，一个进程只能抢一个CPU时间片；
    • 一个地址空间中可以划分出多个线程，在有效的资源基础上，能够抢更多的CPU时间片。
  • CPU的调度和切换：线程的上下文切换必进程要快得多
    • 上下文切换：进程\进程分时复用CPU时间片，在切换之前会保存上一个任务的状态，下次切换回这个任务时，加载这个状态继续运行，任务从保存到再次加载这个过程就是一次上下文切换。
    • 线程更加廉价，启动和退出的速度快，对系统的冲击小。
      $\color{red}{在处理多任务程序时使用多线程比使用多进程更有优势，但是线程并不是越多越好}$，如何控制线程的个数呢？
  1. 文件IO操作：文件IO对CPU的使用率不高，因此可以分时复用CPU时间片，线程的个数 = 2*CPU核心数（效率最高）。
  2. 处理复杂的算法（主要是CPU进行运算，压力大），线程的个数 = CPU核心数（效率最高）。

思路

  常规思路：一层循环枚举x，当 x(x+1)/2 == (n+x)*(n-x+1)/2 时，返回中枢整数x；循环结束返回-1。

单源最短路问题

发表于 2022-11-27 分类于 算法

问题描述

  单源最短路问题是指在给定的有向图 G 中，求解一个数组dis，其中dis[i]表示从源点start 到目标点i的最短路径长度。
  解决单源最短路问题有 Dijkstra 和 Bellman-Ford 两种算法。

Dijkstra 算法

流程

  1. 初始化dis[start] = 0，其余节点的dis值为无穷大；
  2. 找出一个未被标记的、dis[x]最小的节点 x，然后标记节点 x；
  3. 扫描 x 节点的所有出边（x, y, z），若dis[y] > dis[x]+z，则更新dis[y]；
  4. 重复2、3步骤，直到所有节点被标记。

1. 该算法时间复杂度为O(N^2^)，主要瓶颈在第2步查找在全局最小值的过程，可以使用二叉堆对dis进行维护，优化查找过程。

2. Dijkstra算法基于贪心思想，因此只适用于所有边的长度非负的图。

对数组执行操作

题面

给你一个下标从 0 开始的数组 nums ，数组大小为 n ，且由 非负 整数组成。

你需要对数组执行 n - 1 步操作，其中第 i 步操作（从 0 开始计数）要求对 nums 中第 i 个元素执行下述指令：

如果 nums[i] == nums[i + 1] ，则 nums[i] 的值变成原来的 2 倍，nums[i + 1] 的值变成 0 。否则，跳过这步操作。
在执行完 全部 操作后，将所有 0 移动 到数组的 末尾 。

例如，数组 [1,0,2,0,0,1] 将所有 0 移动到末尾后变为 [1,2,1,0,0,0] 。
返回结果数组。

注意 操作应当 依次有序 执行，而不是一次性全部执行。

示例

示例 1：

输入：nums = [1,2,2,1,1,0]
输出：[1,4,2,0,0,0]
解释：执行以下操作：
- i = 0: nums[0] 和 nums[1] 不相等，跳过这步操作。
- i = 1: nums[1] 和 nums[2] 相等，nums[1] 的值变成原来的 2 倍，nums[2] 的值变成 0 。数组变成 [1,4,0,1,1,0] 。
- i = 2: nums[2] 和 nums[3] 不相等，所以跳过这步操作。
- i = 3: nums[3] 和 nums[4] 相等，nums[3] 的值变成原来的 2 倍，nums[4] 的值变成 0 。数组变成 [1,4,0,2,0,0] 。
- i = 4: nums[4] 和 nums[5] 相等，nums[4] 的值变成原来的 2 倍，nums[5] 的值变成 0 。数组变成 [1,4,0,2,0,0] 。
执行完所有操作后，将 0 全部移动到数组末尾，得到结果数组 [1,4,2,0,0,0] 。

题面

  1. 每次可以选择向左或向右；

样例

KMP模式匹配算法

发表于 2022-11-03 更新于 2022-11-04 分类于 算法

KMP算法解决的是主串（s)中模式串（t）的定位问题。
不同于暴力的O(M*N)算法，KMP算法用O(M)的时间处理模式串，用O(N)的时间完成匹配。

定义

Next[i]:    “t中以i结尾的非前缀子串 与 “t的前缀” 能匹配的最长长度。
            即Next[i]=max{j}，其中j<i，且t[i-j+1 ~ i] = t[1 ~ i];当不存在这样的j时，next[i]=0。
f[i]:       “s中以i结尾的子串” 和 “t的前缀”能匹配的最大长度。

模板

应用

求主串中出现模式串的最大个数，要求两个模式串无重叠部分。

思路

正常的KMP算法模式串匹配是可以重叠的，为此我们用动态规划去重。
定义一个val数组，设s[i-m+1 ~ i] == t时，val[i]=1；否则为0。
设dp[i]表示在第i为最后一次出现时的不重叠模式串最大个数。
则

if(i>=m, val[i]) dp(i)=dp[i-m]+1
else if(i< m, val[i]) dp(i)=1
else dp(i)=0

C++ primer-2. 基本类型

发表于 2022-10-26 分类于 C++

C++定义了几种基本的内置类型，如字符、整形、浮点数等。

基本内置类型

基本内置类型包括算数类型和空类型(void)，算数类型包括字符、整数型、浮点数和布尔值。

算数类型

算数类型分为整数型和浮点型两类。字符和布尔值都属于整型。
C++定义了各类型的最小尺寸：bool未定义，char 8位，short 16位，int 16位，long 32位，long long 64位；
float是32位4字节，包括6位有效数字，double是64位8字节，包括10位有效数字。

int、short、long都是带符号类型，char由编译器决定是否有符号。
C++提供了几种字符类型：

  1. char：一个char的空间应确保可以存放机器基本字符集中任意字符对应的数字值，即一个char的大小和一个机器字节一样。
  2. wchar_t：宽字节，用于拓展字符集，wchar_t确保可以存放机器最大扩展字符集中的任意一个字符。
  3. char16_t和char32_t：为Unicode字符集服务。

类型选择：

  • 明确数值不可能为负时，选用无符号类型。
  • 整数运算用int，数值太大用long long，不用short和long
  • 浮点数运算用double。float和double的计算代价相差无几。
