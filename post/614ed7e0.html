合并两个二维数组-求和法

题目描述

给定两个二维整数组 A 和 B，A[i]={x, y}表示有 y 个 x ，B同理。
两个数组的 x 不一定相同，但一定按递增排序。
要求合并两个数组得到数组C，要求 C 中的也是递增排序；且每个 x 在 C 中只出现一次，x 对应的个数y为A、B中的个数和。

思路

  1. 简单用一个map维护，时间复杂度为O(M+N)，空间复杂度为O(M+N)；
  2. 归并排序，时间复杂度同上，空间优化到O(1)

代码：归并

将整数减到零的最少操作次数

题目描述

给定一个正整数n(<1e5)，可以执行以下操作任意次：

  • n 加上或减去 2的某个幂(1,2,4,8,16……)；
    返回使 n 等于 0 的最少执行次数。

思路

消去1的方式无非两种：减去1；加上1，使得当前位为0，高位连锁反应。因此我们应该从低位向高位计算

  1. 贪心：考虑二进制中连续1的个数x。可以推导：如果x大于2，可以用一次加法和一次减法置零这一段；考虑示例2中的数字(54)，可以适当保留加法进位的1。时间复杂度O(logN)。
  2. 记忆化搜索，编码简单，时间和空间都消耗更大。

无平方子集计数

题目描述

给你一个正整数数组 nums 。

如果数组 nums 的子集中的元素乘积是一个无平方因子数 NSQ，则认为该子集是一个无平方子集。

无平方因子数是无法被除 1 之外任何平方数整除的数字。

返回数组 nums 中无平方且非空的子集数目。因为答案可能很大，返回对 1e9+7 取余的结果。

nums 的非空子集是可以由删除 nums 中一些元素（可以不删除，但不能全部删除）得到的一个数组。如果构成两个子集时选择删除的下标不同，则认为这两个子集不同

思路

已知 nums[i]<30，nums.length<1000。
率先想到质数，小于30的质数有10个；
我们可以将 30 以内的NSG对应的质因子集合表示成一个二进制数mask[i]；注意4、9这种本身即平方的数mask[i]置为-1；在本题中mask[i]显然<210；
我们需要实现每次选取一个数字 x 加入到已有的集合 y 中，要满足 mask[x] & mask[y] == 0；如果不为0则出现了平方因子数。
求这种选取的方案数，可以考虑到使用01背包。

找出对应LCP矩阵的字符串

题目描述

对任一由 n 个小写英文字母组成的字符串 word ，我们可以定义一个 n x n 的矩阵，并满足：

  • lcp[i][j] 等于子字符串 word[i,…,n-1] 和 word[j,…,n-1] 之间的最长公共前缀的长度。
    给你一个 n x n 的矩阵 lcp 。返回与 lcp 对应的、按字典序最小的字符串 word 。如果不存在这样的字符串，则返回空字符串。

思路

不难想到的两个步骤:构造和验证。
在比赛中，已经成功构造了符合要求的字符串，但没有想到用dp来进行验证。
可以预想到可能出现的问题：

  • 字符种类超过26个（在构造时解决）
  • 在 s[i]!=s[j] 时 lcp[i][j]!=0（在验证时解决）
  • lcp不满足递增，即在s[i]==s[j]时，lcp[i][j]==lcp[i+1][j+1]。（在验证时解决）
