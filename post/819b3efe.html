最长平衡子字符串

思路：
能暴力绝不优化，模拟！
开个玩笑哈哈哈！O(N2) 的时间复杂度还是太高了，我们可以考虑记录连续的 0 的个数pre 和 1 的个数 p，这个 01 串的长度就是 2*min(pre, p)。取所有长度的最大值。
一次遍历即可完成。

思路：

  • 用一个哈希表存每个元素出现的次数。
  • 构造答案时，如果元素 x 的剩余次数为0，则从哈希表中删除。

老鼠和奶酪

思路：

  • 我们记录一个中间数组 t，使得 t[i]=r1[i]-r2[2];
  • ans += r2[i]；这步操作意味着一号老鼠没有吃奶酪。
  • 接着我们排序数组 t，然后将最大的前 k 个 t[i] 加在ans上，这一步意味着把二号吃的吐出来交给一号。
  • 一个小小的贪心算法，不多解释。

深搜思路

思路不算太难，主要是混合的知识点不少：

  1. 不难想到深搜。但深搜的队列节点有n个，对于每个节点翻转后的下一个节点，如果不进行优化，其可能性也有O(N)。即时间复杂度为O(N2)，但 n 最大值为 1e5。
  2. 如何优化翻转后的节点的遍历？思路第一步检查了很多没不要的翻转（已经经过了），可以考虑使用一个数据结构，每当某个节点已经经过了，则从该数据结构中删除，这样总的遍历次数也就 n 次，没有多余的重复遍历。赛中的时候到这一步就没有思路了，set是一个很合适的数据结构，其删除O(1)，查找O(logN)。
  3. 分类的考虑。对于位置 i 上的 1，翻转之后的所有位置是一个公差为 2 的等差数列（不考虑banned）。因此我们按奇偶分类set，这样才能保证翻转时的合法性。
  4. 边界的考虑。编码过程中发现，可以翻转的区间这一细节的实现比较容易出错。它并不是简单的 L=max(0, now-k+1)，r=min(n-1, now+k-1)。当 now < k 时，受到数组左边界的约束，小于 k-now-1 的点是无法翻转的。例如 now=3，k=5；此时 0 号位就永远无法翻转到。
