个人想法

个人感觉模板四是最好入门的，毕竟大部分程序员把二分写进死循环的原因都是对边界的判断不对。那么所幸我们把边界放大，让最终结果一定在[l, r]中，最后根据题目需求从左到右或者从右到左遍历。同时我们限定的范围也不大，对时间复杂度的影响也是微乎其微的。

实战

1. 两球之间的磁力

题目描述

给出x轴上n个篮子的位置，你手上有m个小球，要求小球必须放在篮子里。a、b两处的小球之间的磁力为|a-b|。要求最大化的最小磁力。
2 <= m <= n <= 1e5;
1 <= pos[i] <= 1e9;
保证任意pos中的数互不相同;

样例

pos = {1,2,3,4,7}, m = 3;
ans = 3;

pos = {5,4,3,2,1,1000000000], m = 2;
ans = 999999999;

具体解释可以点击题目查看原题。

思路

1. 先排序pos。
2. 按照常规思路用二分去搜索一个磁力值mid，在check()函数中，我们利用贪心的思路：判断当前篮子距离上个小球的距离是否>mid，依此是否放入小球。
3. check(): 放入的小球总数 > mid，说明这个磁力值 mid 不能实现，return 0。否则return 1。

2. 袋子里最少数目的球

题目描述

给定 n 堆，每堆有 a[i] 个球，你可以操作 m 次，每次可以把一堆球分为两堆。要求最小化最大堆的球数量并输出。

样例

a = [9], m = 2;
ans = 3。

a = [2,4,8,2], m = 4;
ans = 2;

思路

1. 二分最大堆球数 mid，如果check(mid)为真，r = m-1。 否则 l = m+1。
2. check(x, m): 验证能否在 m 次操作内把原来 >x 的球堆分开。
3. op += (a[i]-1) / x;
